Hackathon orchestration

workflowWrapper

A thin Bash CLI (wfw) that routes commands to treehouse lavish-axi gnhf no-mistakes without bundling their source.

Install
npm link
from this repo root
Entry
bin/hack-wrap.sh

First-time setup (install once per machine)

Before wfw start, each teammate needs these on PATH:

  • wfw - npm link from this repo
  • treehouse - worktree pool manager
  • gnhf - autonomous coding agent
  • node + npm - for npx lavish-axi (plan command)
  • no-mistakes - git remote named no-mistakes on your repo

Command flow

flowchart TD
  U["Developer"] --> WFW["wfw CLI"]
  WFW -->|start| TH["treehouse init / get --lease"]
  WFW -->|plan| LV["npx lavish-axi lavish_artifact.html"]
  WFW -->|auto| GN["gnhf objective"]
  WFW -->|validate| NM["git push no-mistakes HEAD"]
  TH --> WT["Feature worktree"]
  WT --> SYM["lavish_artifact.html symlink"]
  SYM --> SP["my_team_workspace/shared_lavish_plan.html"]

wfw start <feature>

  • Create my_team_workspace/ if missing
  • Touch empty shared_lavish_plan.html
  • treehouse init when no treehouse.toml
  • treehouse get --lease --lease-holder <feature>
  • Symlink plan into worktree as lavish_artifact.html
  • Append symlink to .git/info/exclude
Absolute symlink path used because treehouse pools worktrees outside the workspace folder.

Other commands

CommandGuardDelegates to
wfw plan [prompt] optional lavish-axi; with prompt = /lavish-style build
wfw auto "..." worktree gnhf --max-iterations 12 --max-tokens 30000
wfw validate worktree git push no-mistakes HEAD
wfw treehouse ... passthrough full treehouse CLI
wfw lavish ... passthrough full lavish-axi CLI
wfw gnhf ... guarded gnhf with same guardrails

Worktree detection: lavish_artifact.html exists and cwd is inside a git work tree.

gnhf guardrails (wfw auto and wfw gnhf)

Always passes --max-iterations 12 and --max-tokens 30000.

Override via WFW_GNHF_MAX_ITERATIONS / WFW_GNHF_MAX_TOKENS.

Terminal + LLM slash commands

Download via npm link + npm run install-skill. Agents get /wfw start, /wfw prompt, /wfw auto, etc.

wfw plan "..." queues a Lavish build like /lavish on the shared plan artifact.

Filesystem layout

./my_team_workspace/
├── treehouse.toml
├── shared_lavish_plan.html   # team-wide Lavish plan
└── (treehouse pool elsewhere)
    └── <leased-worktree>/
        ├── lavish_artifact.html -> abs path to shared plan
        └── .git/info/exclude    # ignores symlink

{

}

Quick start

cd optimizeWrkFlow
npm link
wfw start my-feature
wfw plan

Risks and assumptions

no-mistakes remote

Repo must already have no-mistakes git remote configured.

treehouse pool

init runs inside my_team_workspace; leased paths come from treehouse stdout.

Shared plan

All feature worktrees symlink to one HTML file for collaborative Lavish planning.
